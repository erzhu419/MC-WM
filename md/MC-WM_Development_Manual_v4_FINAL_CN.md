# MC-WM：元认知世界模型与全元组残差动力学

## 开发手册 v4 — 最终版

---

## 1. 项目定位

**标题：** *通过符号动力学发现与残差 Bellman 方程实现外推式残差世界模型的仿真到现实迁移*

**一句话概括：** 我们用全元组符号残差模型修正仿真动力学，通过 NAU/NMU 实现外推，利用残差 Bellman 方程（考虑多步累积误差）对纠正进行门控，并引入 LLM 作为元认知模块——同时发现残差中的隐藏结构，并维护一个单调增长的物理与语义约束集合。

**基础原则：** *每一个关于仿真-现实差距的新假设都必须满足所有已建立的约束。约束集合只增不减——正如科学一样，新理论必须解释旧理论能解释的一切，且能解释更多。*

### 2.1 LLM 的三种角色

| 角色 | 时机 | 功能 | 频率 |
|---|---|---|---|
| #1 约束初始化 | 第 0 阶段，训练前 | 根据环境描述生成物理和语义约束 | 一次 |
| #2 特征假设 | 训练时，在自我假设循环内 | 在 4 种自动机制失败时提出新特征 | 较少（累计 20-60 次调用） |
| #3 约束增强 | 训练时，每 N 个 epoch | 审核可疑纠正，生成新约束 | 周期性（累计 10-30 次调用） |

角色 #1 回答："这个世界中什么是不可能的？"  
角色 #2 回答："什么隐藏模式可以解释这个残差？"  
角色 #3 回答："这通过了我所有的规则，但看起来不对——我遗漏了什么规则？"

| # | 组件 | 移除后… |

## 3. 全元组残差

### 3.1 定义

$$\Delta(s,a) = \begin{pmatrix} s'_{\text{real}} - s'_{\text{sim}} \\ r_{\text{real}} - r_{\text{sim}} \\ d_{\text{real}} - d_{\text{sim}} \end{pmatrix}$$

每个元素都有自己的 SINDy 模型、NAU/NMU 头和 Q_Δ 分量。

### 3.2 为何需要每个元素

仅纠正状态会遗漏：错误的奖励（→ 错误的价值估计）、错误的终止（→ 错误的时间跨度）、错误的噪声水平（→ 虚假的置信度）。示例：带软地毯的 Ice-Walker——状态动力学改变，同时跌倒检测阈值改变，电机能量成本也改变。仅纠正状态仍会学到错误的策略。

### 3.3 维度不匹配

如果仿真有 17 维而真实有 15 维：LLM 角色 #1 在第 0 阶段生成维度映射规则。要么丢弃仿真专有维度，要么从物理耦合中推断缺失的真实维度。映射在训练前冻结。

## 4. 自我假设循环

### 4.1 循环流程

```
FOR round = 1 to max_rounds:

    假设：在当前基函数库上拟合 SINDy
    测试：留出集质量门控（导数误差 < ε_threshold）
    IF 通过 → 接受符号模型，完成

    证伪：对余项运行诊断电池
    IF 所有检验为负 → 余项是真正的偶然不确定性，完成

    扩展（自动化，4 种机制）：
        自相关阳性 → 添加时间延迟特征（Takens）
        异方差阳性 → 添加罪魁变量的非线性项
        重尾 → 添加分段接触掩码
        非平稳 → 添加时间特征

    IF 自动扩展添加了新特征 → CONTINUE 进入下一轮

    扩展（LLM 角色 #2）：                    ← 仅在自动扩展耗尽后
        将诊断报告发给 LLM
        LLM 提出候选特征
        通过 ASTEval 安全求值
        将有效特征添加到 SINDy 库
        IF 新 SINDy 拟合通过质量门控 → CONTINUE
        ELSE → 接受当前最优模型，完成
```

### 4.2 四种自动扩展机制

**机制 1 — Takens 时间延迟嵌入**
- 触发条件：在滞后 k 处自相关阳性
- 动作：添加 $s_{t-1}, s_{t-2}, (s_t - s_{t-1})/dt$
- 处理：隐藏风力、电机延迟、负载变化

**机制 2 — 代数特征交叉**
- 触发条件：异方差阳性，罪魁变量 = $j$
- 动作：添加 $s_j^2, s_j|s_j|, s_j^3, s_j \cdot a_k$
- 处理：二次阻力、非线性摩擦

**机制 3 — 分段逻辑掩码**
- 触发条件：重尾残差（峰度 > 4）
- 动作：对 |残差| 做 k-means（k=2），添加 $\mathbb{1}(s_j < \text{threshold})$
- 处理：接触/飞行模式切换

**机制 4 — 时间特征**
- 触发条件：平稳性检验阳性
- 动作：添加归一化步数索引、累积量
- 处理：时变差距（磨损、温度漂移）

### 4.3 LLM 特征假设（角色 #2）

仅在所有 4 种自动机制都无法改善质量门控时激活。

```
系统：你是一名物理特征工程师。
可观测变量：{obs_keys}。历史：obs_prev。动作：{action_keys}。
规则：只使用现有变量 + numpy 数学运算。不得发明传感器。

用户：残差维度 {name}。诊断：{report}。
已尝试：{自动扩展失败特征列表}。
我遗漏了什么特征？
```

LLM 提出的特征与自动扩展特征经过相同的质量门控，没有特殊待遇。

### 4.4 输出

1. **每个元组元素的符号残差模型** → NAU/NMU 头
2. **偶然性余项**（诊断确认无结构）→ 鲁棒强化学习不确定性集合

## 5. NAU/NMU 外推头

SINDy 发现符号形式，NAU/NMU 在可微网络中强制执行，并保证外推性：

$$\|\hat{\Delta}(s_{\text{ood}},a) - \Delta_{\text{true}}(s_{\text{ood}},a)\| \leq \epsilon + \varepsilon\|d\| + \frac{L}{2}\|d\|^2$$

- NAU：$L = 0$（线性外推，误差界最紧）
- NMU：$L = 2|c|$（二次外推）
- ReLU：$L = \infty$（不存在误差界——定理 4.12）

## 6. 残差 Bellman 方程

### 6.1 问题

逐转换纠正忽略了累积效应。步骤 $t$ 的误差会移动步骤 $t+1$ 的状态，进而移动 $t+1$ 处的纠正，在整条轨迹上不断累积。这是纠正质量的信用分配问题。

### 6.2 定义

$$Q_\Delta(s,a) = \underbrace{\|\hat{\Delta}(s,a) - \Delta_{\text{true}}(s,a)\|^2}_{\text{当前纠正误差}} + \gamma \cdot \mathbb{E}_{a' \sim \pi}\left[Q_\Delta(s'_{\text{corrected}}, a')\right]$$

"奖励" = 纠正误差。Bellman 递推将未来累积误差向后传播到当前状态。

### 6.3 门控

$$g(s,a) = \sigma\left(\frac{\tau - Q_\Delta(s,a)}{\text{temperature}}\right)$$

低 Q_Δ → 纠正在整条轨迹上可靠 → 门控开启。  
高 Q_Δ → 纠正会严重累积 → 门控关闭 → 回退到原始仿真。

### 6.4 训练

在离线配对数据上，真实纠正误差已知。使用拟合 Q 迭代，当纠正后下一状态的 Q_Δ 不确定时采用悲观估计。

## 7. 约束系统

### 7.1 单调增长原则

约束集合 $\mathcal{C}$ 只增不减：

$$\mathcal{C}_0 \subseteq \mathcal{C}_1 \subseteq \mathcal{C}_2 \subseteq \ldots$$

新约束被添加进去，旧约束永远不会被移除。这反映了科学知识的结构：新理论必须满足所有已建立的定律。

任何违反 $\mathcal{C}$ 中任意约束的纠正都会被拒绝。随着 $\mathcal{C}$ 增长，可行纠正空间收缩，变得越来越精确。

### 7.2 第 0 阶段：初始约束生成（LLM 角色 #1）

LLM 根据环境描述生成约束。三种类型：

**物理可能性：**
```json
{"name": "joint_limit", "expr": "abs(obs_next['3']) > 3.14"}
{"name": "underground", "expr": "obs_next['1'] < -0.3"}
```

**语义合理性：**
```json
{"name": "robot_flying", "expr": "obs_next['1'] > 2.0"}
{"name": "impossible_acceleration", "expr": "abs(obs_next['9'] - obs['9']) / dt > 100"}
```

**维度映射（如果仿真 ≠ 真实维度）：**
```json
{"type": "drop", "dims": [15, 16], "reason": "sim-internal variables"}
```

人工审核。对真实数据的误拒率 < 1%。冻结为 $\mathcal{C}_0$。

### 7.3 训练阶段：约束增强（LLM 角色 #3）

**为何第 0 阶段不够：** 第 0 阶段约束仅根据环境描述生成，未见实际数据。LLM 无法预见每个边缘案例。训练开始后，仿真探索多样化状态，会出现第 0 阶段未覆盖的新情况。

**触发条件：** 每 N 个 epoch，收集"可疑纠正"：
- 所有现有约束 $\mathcal{C}_k$ 通过（未被当前规则捕获）
- 且纠正幅度 > 中位数的 2 倍（异常大）
- 且 Q_Δ 处于不确定区域（不明显安全，也不明显危险）

**流程：**
```
将可疑纠正批量发给 LLM：
  "这 10 个纠正后的转换通过了我所有的约束。
   其中有没有物理上不可能或语义上不合理的？
   对每个有问题的，给我一条新的约束规则。"

LLM 回应：
  "转换 3：机器人倒置（rootangle = 2.8 rad）。
   新约束：obs_next['rootangle'] > 2.5 or obs_next['rootangle'] < -2.5"

  "转换 7：前向速度 45 m/s 对这个机器人不合理。
   新约束：abs(obs_next['rootx_vel']) > 30"

新约束 → 人工可选择性审核（异步）→ 添加到 C_{k+1}
```

**关键特性：** 已在增强缓冲区中的现有纠正**不会**被追溯性移除。新约束只适用于未来的纠正。这防止了训练期间缓冲区不可预测地缩小。

### 7.4 运行时应用

```python
def passes_constraints(s, a, s_corrected, constraint_set):
    for c in constraint_set:
        if eval_constraint(c, s, a, s_corrected):
            return False  # 违反 → 拒绝
    return True  # 全部通过 → 接受
```

### 7.5 为何不用判别器

| | 判别器 | LLM 约束 |
|---|---|---|
| 判断 | "统计上正常？" | "物理上可能？语义上合理？" |
| 关节角度合法的倒置机器人 | 可能通过（统计上合理） | 失败（"机器人不应该倒置"） |
| 依赖 | 训练数据（OOD = 不可靠） | 世界知识（OOD = 仍然可靠） |
| 可解释性 | 无 | 有（布尔规则，可审计） |
| 随时间增长 | 否（训练后固定） | 是（单调增强） |

## 8. 策略学习

### 8.1 增强缓冲区构建

```
对每个仿真转换 (s, a)：
    1. 计算符号纠正：Δ̂_s, Δ̂_r, Δ̂_d
    2. 应用纠正：s' = s'_sim + Δ̂_s, r' = r_sim + Δ̂_r
    3. 检查约束：如果有违反 → 拒绝，跳过
    4. 计算 Q_Δ 门控：g = σ((τ - Q_Δ(s,a)) / temp)
    5. 最终：s' = s'_sim + g·Δ̂_s, r' = r_sim + g·Δ̂_r
    6. 如果 g > 阈值 → 以 confidence = g 添加到缓冲区
```

所有真实数据以 confidence = 1.0 添加。

### 8.2 鲁棒策略

带置信度加权 Critic 损失的 IQL/CalQL：
- 高置信度（g ≈ 1）：标准损失
- 低置信度（g ≈ 0）：与偶然方差成比例的悲观惩罚

## 9. 理论保证

| 定理 | 陈述 | Lean 4 预估行数 |
|---|---|---|
| 逐元素 OOD 误差界 | $\|\hat{\Delta}_e - \Delta_e^{\text{true}}\| \leq \epsilon + \varepsilon\|d\| + \frac{L}{2}\|d\|^2$ | 继承自 CS-BAPR |
| Q_Δ 压缩性 | 在冻结残差模型下，Q_Δ 算子是 γ-压缩的 | 约 80 行，新颖 |
| 门控安全性 | g → 0 当 Q_Δ 大时 → 回退到原始仿真，不变差 | 约 40 行 |
| 单调改进 | 假设循环每一轮通过质量门控都收紧 ε | 约 100 行 |
| 约束收紧 | diam(C) < OOD_bound → 约束胜出 | 约 40 行 |
| 约束单调性 | C_k ⊆ C_{k+1} → 可行纠正集只缩小 → 安全性只提升 | 约 20 行，平凡 |

## 10. 实验

### 10.1 HP-MuJoCo 基准

| 环境 | 状态差距 | 奖励差距 | 终止差距 |
|---|---|---|---|
| Aero-Cheetah | 二次阻力 $-kv^2$ | 能量成本 | — |
| Ice-Walker | $x>5$ 处摩擦下降 | 速度缩放 | 更软的跌倒阈值 |
| Wind-Hopper | 正弦侧风 | — | 风致跌倒 |
| Carpet-Ant | 阻尼接触 | 电机电流 | 软跌倒 |

### 10.2 基线方法

H2O+（重要性采样）、DARC（奖励增强）、IGDF（对比滤波）、ReDRAW（NN 残差）。

### 10.3 关键实验

1. **全元组 vs 仅状态：** Ice-Walker — 错误的终止杀死了仅状态纠正的智能体
2. **OOD 外推：** 1x → 8x 速度 — 我们的方法呈多项式增长，ReDRAW 崩溃
3. **Q_Δ vs 距离门控：** Q_Δ 对"远但建模良好"开放，对"近但危险"关闭
4. **自我假设消融：** 逐轮改进，自动扩展 vs +LLM
5. **约束增强：** 展示新约束捕获了第 0 阶段遗漏的案例
6. **LLM 特征假设：** 展示 LLM 发现了自动扩展无法发现的跨维度特征

## 12. 贡献

1. **全元组残差世界模型。** 纠正状态、奖励、终止——而非仅状态。
2. **自我假设符号发现。** 自动诊断 → 扩展 → SINDy，LLM 备选用于超出自动机制能力的特征。
3. **残差 Bellman 方程。** 轨迹感知的纠正质量评估，取代临时设计的门控。
4. **可外推残差（NAU/NMU）。** 正式的 OOD 多项式误差界，解决 ReDRAW 的局限。
5. **单调增长约束集合。** LLM 生成初始约束并在训练期间增强，新假设满足所有过去的约束。

---

## 13. 局限性

- **set_state() 假设：** 仿真必须可以在任意 (s,a) 处查询。
- **SINDy 表达能力：** 差距必须在某个基函数上稀疏。涵盖参数不匹配；不涵盖完整引擎差异。
- **Q_Δ 自举：** 悲观估计可以缓解但无法消除远 OOD 区域的累积误差。
- **LLM 约束质量：** 取决于 LLM 对特定领域的世界知识。通过人工审核来缓解。
